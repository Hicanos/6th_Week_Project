## Commit Convention
> Add:
파일 추가
> 
> Remove :
파일 삭제
> 
> Rename :
파일 혹은 폴더명 수정 혹은 이동
> 
> Feat :
새로운 기능 추가
> 
> Fix : 
버그 수정
> 
> Docs : 
문서 수정
> 
> Style : 
코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
> 
> Refactor : 
코드 리펙토링
> 
> Test : 
테스트(테스트용 디버그 코드 추가, 수정, 삭제 등 기본 로직에 변경이 없는 경우)
> 
> Chore : 
위에 걸리지 않는 기타 변경사항   
> 
> Comment : 
필요한 주석 추가 및 변경   
> 
> Init : 
프로젝트 초기 생성
